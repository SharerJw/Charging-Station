# 🔧 充电平台 - 产品运维（App）深度细化提示词

针对**产品运维（App）**，我将其拆分为 **7个核心模块** 进行深度细化。这些提示词涵盖了一线运维工程师从接单、出勤、现场操作、远程管控到交接班的全链路业务场景，可直接输入给 AI 编码工具生成高质量代码。

## 提示词 1：全局架构、登录鉴权与多角色工作台

> **定位**：App 的基础骨架、登录流程、多角色适配（运维工程师/站长/区域经理）以及个性化工作台。

```text
请生成充电桩运维App的全局架构、登录鉴权模块和多角色工作台。
技术栈：UniApp + Vue 3 (Composition API) + TypeScript + Pinia + uView Plus。
目标平台：iOS 13+、Android 8+，适配主流刘海屏/挖孔屏/折叠屏。
UI风格：专业工具风，主色调 品牌蓝(#1677FF)，背景色 #F5F7FA，卡片白(#FFFFFF)，圆角 12px，阴影轻柔。

【全局架构设计】
1. 底部 TabBar（5个Tab，图标+文字，选中态蓝色）：
   - 🏠 工作台 | 📋 工单 | 🔍 巡检 | 💬 消息 | 👤 我的
   - 工单 Tab 显示待处理数量红色角标，消息 Tab 显示未读数角标。
2. 全局顶部导航栏：
   - 左侧：当前定位城市 + 站点名称（如在站点范围内自动识别）。
   - 右侧：扫一扫图标（核心入口）、消息铃铛（带角标）。
3. 全局组件：
   - 网络状态横幅：断网时顶部显示红色横幅"网络连接已断开，当前为离线模式"。
   - GPS 精度提示：当定位精度>50m时，显示黄色提示"定位精度较低，可能影响到站验证"。
   - 全局 Loading 遮罩：带品牌 Logo 旋转动画。

【登录鉴权模块】
1. 启动页（Splash Screen）：
   - 品牌 Logo + "运维通" App 名称 + Slogan "智慧充电，高效运维"。
   - 底部：公司 Logo + 版本号，持续 2 秒自动跳转。

2. 登录页：
   - 顶部：品牌 Logo + 欢迎标题。
   - 登录方式Tab切换：
     * 账号密码登录：手机号/工号输入框（带清除按钮）+ 密码输入框（明文切换图标）。
     * 验证码登录：手机号输入 + 获取验证码按钮（60秒倒计时）+ 验证码输入。
   - 生物识别快捷登录（如已绑定）：指纹/面容登录按钮（大图标，一键触发）。
   - 底部：[登录] 大按钮（蓝色渐变）+ "忘记密码"链接 + "遇到问题"帮助链接。
   - 安全提示：底部小字"本App仅限授权运维人员使用，登录行为将被审计记录"。

3. 首次登录/新设备绑定流程（登录后触发）：
   - Step 1：选择所属区域/运维团队（如已预分配则跳过）。
   - Step 2：绑定工牌 NFC / 扫描工牌二维码（用于后续现场身份核验）。
   - Step 3：录入人脸信息（用于敏感操作二次验证，调用活体检测 SDK）。
   - Step 4：权限与通知授权引导（定位、相机、通知、NFC）。

【多角色工作台（首页）】
根据登录用户角色动态渲染不同的工作台内容：

角色A - 一线运维工程师视角：
   1. 今日概览卡片（横向滚动，3张卡片）：
      * "今日工单"：待处理 X 件 / 已完成 Y 件（带环形进度图）。
      * "今日巡检"：待巡检 X 站 / 已完成 Y 站。
      * "紧急告警"：当前未处理告警 X 条（红色高亮，点击跳转告警列表）。
   2. 我的任务看板（按优先级排序的任务列表，显示最近5条）：
      * 每条任务卡片：优先级色条(左边缘) + 工单编号 + 站点名 + 故障摘要 + 倒计时 + 距离。
   3. 快捷工具栏（2×4 图标网格）：
      * 扫码操作 | 一键报修 | 远程重启 | 设备台账
      * 备件领用 | 知识库 | 交接班 | 更多
   4. 实时告警流（最新3条，带闪烁动画）：
      * 告警级别图标 + 站点名 + 告警摘要 + 时间 + [接单]按钮。

角色B - 站长视角：
   1. 我的站点总览卡片：
      * 站点名称、设备总数、在线率(%)、今日充电量(kWh)、今日营收(元)。
      * 设备状态分布小饼图（在线/离线/故障）。
   2. 站点排名（与其他站点对比的关键指标排行）。
   3. 下属运维人员状态列表（头像+姓名+当前状态[空闲/出勤中/休息]+今日完成工单数）。
   4. 快捷操作：派发工单、安排巡检计划、站点公告。

角色C - 区域经理视角：
   1. 区域数据大盘（mini版数据大屏）：
      * 管辖站点数、设备总数、整体在线率、今日营收、工单完成率。
   2. 异常站点 Top5 列表（按故障率/离线率排序）。
   3. 团队效率指标：平均响应时间、平均修复时间(MTTR)、工单积压数。

要求：
- 登录 Token 使用 JWT，支持自动刷新（Refresh Token机制），过期自动跳转登录页。
- 支持多账号切换（"切换账号"功能，保留最近3个登录账号）。
- 工作台数据支持下拉刷新，卡片数据懒加载。
- 离线时工作台显示最后一次同步的数据快照 + "数据更新于 XX 分钟前"提示。
```

## 提示词 2：工单全生命周期管理（核心业务模块）

> **定位**：运维App的核心业务。覆盖工单的创建、接收、响应、到场、处理、验收、关闭的全生命周期，以及工单转派、协同、SLA 超时预警等高级功能。

```text
请生成充电桩运维App的"工单全生命周期管理"模块，包含工单列表页、工单详情页、工单处理页和工单统计页。
技术栈：UniApp + Vue 3 + Pinia + uView Plus。
UI风格：效率优先，信息层级清晰，关键操作按钮大而醒目。

【页面一：工单列表页（TabBar第二Tab）】
1. 顶部筛选区：
   - Tab 切换：待接单(3) | 处理中(2) | 待验收(1) | 已完成 | 全部
   - 筛选抽屉（点击"筛选"图标从底部弹出）：
     * 优先级：P0紧急 / P1高 / P2中 / P3低（多选标签）
     * 工单类型：故障维修 / 定期巡检 / 设备安装 / 设备搬迁 / 用户投诉 / 预防性维护
     * 时间范围：今天 / 本周 / 本月 / 自定义日期区间
     * 站点选择（下拉树形选择器：区域→站点）
     * 排序方式：按创建时间 / 按SLA剩余时间 / 按距离

2. 工单卡片列表（每项为一张卡片，带左侧优先级色条）：
   - 优先级色条：P0=红色(4px宽) / P1=橙色 / P2=蓝色 / P3=灰色
   - 卡片头部：
     * 工单编号（如 "WO-20240713-0089"，可长按复制）
     * 工单类型 Tag（如 "故障维修"，圆角标签）
     * SLA 倒计时（如 "剩余 1h 23min"，<30min 时红色闪烁）
   - 卡片主体：
     * 站点名称（大字号加粗）+ 距离（如 "2.3km"，绿色文字）
     * 设备编号 + 设备型号（如 "CP-A023 | 120kW DC"）
     * 故障摘要（1行截断，如 "充电枪A无法锁紧，用户反馈..."）
     * 故障码 Tag（如 "ERR_LOCK_003"，点击可查看知识库解释）
   - 卡片底部：
     * 创建时间 + 报修人
     * 操作按钮（根据状态动态显示）：
       - 待接单状态：[接单]（主按钮蓝色） + [转派]（次按钮）
       - 处理中状态：[前往处理]（主按钮）
       - 待验收状态：[查看进度]（主按钮）

3. 列表交互：
   - 左滑卡片：显示 [转派] [催办] [备注] 快捷操作按钮。
   - 下拉刷新 + 上拉加载更多（分页，每页 20 条）。
   - 空状态插画："暂无工单，享受片刻清闲 ☕"
   - 列表顶部浮动筛选胶囊：已激活的筛选条件标签，可点X移除。

【页面二：工单详情页（点击卡片进入）】
1. 顶部区域：
   - 导航栏：返回箭头 + 工单编号 + 右上角 "..." 更多菜单（分享/催办/投诉/复制工单号）。
   - 工单状态大标签：全宽横幅显示当前状态 + 状态色背景（如 "🔴 P0 紧急 | 处理中"）。

2. SLA 进度条（横向时间轴）：
   - 节点：创建(09:15) → 接单(09:18) → 出发(09:25) → 到场(09:42) → 处理中 → 完成
   - 已完成节点：蓝色实心圆 + 时间。
   - 当前节点：蓝色脉冲动画圆点。
   - 未到达节点：灰色空心圆。
   - 时间轴下方：SLA 倒计时进度条（绿→黄→红渐变，剩余时间<30%时变红）。

3. 信息卡片组（折叠面板形式）：
   - 📍 站点与位置信息（可展开/折叠）：
     * 站点名称、详细地址（点击跳转第三方地图导航）。
     * 站点联系人 + 电话（一键拨号按钮）。
     * 站点照片缩略图（点击放大查看，了解现场环境）。
     * [导航前往] 大按钮（调用高德/百度/腾讯地图，自动选择最快路线）。
   - 🔌 设备信息：
     * 设备编号、型号、固件版本、安装日期。
     * 设备当前状态（在线/离线/故障，带状态色点）。
     * 最近 5 条告警记录（摘要列表，点击展开详情）。
     * 设备历史维修次数 + 最近一次维修时间。
     * [查看设备档案] 链接 → 跳转设备台账详情。
   - 📝 故障/任务描述：
     * 详细描述文本（支持富文本/图文混排）。
     * 报修人上传的故障照片/视频（九宫格展示，点击查看大图）。
     * 故障码说明 + 关联知识库文章推荐（如 "点击查看：ERR_LOCK_003 常见处理方案"）。
   - 👤 报修人信息：
     * 姓名、电话（一键拨号）、角色（用户/站长/系统自动）。

4. 操作区域（根据工单状态动态渲染）：
   
   状态=待接单：
   - [✅ 接受工单] 大按钮（蓝色，底部固定）。
   - [↗ 转派他人] 按钮（选择转派对象弹窗：按距离/按技能/按工作量推荐人员列表）。
   
   状态=已接单/前往中：
   - [📍 确认到场] 大按钮（点击后调用 GPS 验证：当前位置与站点距离<100m 方可确认，否则提示"距离站点还有X.Xkm，确认提前签到？"）。
   - [导航前往] 次按钮。
   - 预计到达时间输入（反馈给调度中心）。
   
   状态=处理中（核心操作页面，另开子页面）：
   见下方【页面三：工单处理页】。
   
   状态=待验收：
   - 显示处理结果摘要 + 照片。
   - [✅ 确认验收] / [❌ 退回重处理] 按钮。
   
   状态=已完成：
   - 完整处理报告展示。
   - [📄 导出报告] / [⭐ 评价协作方] 按钮。

5. 底部时间线（操作记录/沟通记录混合展示）：
   - 竖向时间轴，每条记录包含：
     * 操作人头像 + 姓名 + 角色标签。
     * 动作描述（如 "张工 接受了工单"、"系统 SLA预警：剩余30分钟"、"李工 添加了备注"）。
     * 时间戳。
     * 附件缩略图（如有拍照/录音）。
   - 底部输入区：文本输入框 + 📷拍照 + 🎤语音 + 📎附件 按钮，可发送沟通消息（类似工单内聊天）。

【页面三：工单处理页（处理中状态的核心操作）】
1. 到场确认区：
   - [📱 扫描设备二维码/NFC] 按钮：扫描充电桩机身二维码或贴近NFC标签，验证设备身份（防止修错设备）。
   - 验证成功后显示绿色对勾 + 设备信息确认卡片。

2. 故障诊断辅助：
   - AI 智能诊断入口（如有）：上传故障照片/描述，AI 推荐可能原因和解决方案。
   - 关联知识库推荐列表（根据故障码自动匹配）。
   - 远程诊断工具入口：[查看实时数据] / [远程重启] / [远程升级]（跳转远程控制页）。

3. 处理记录表单（必填项标红星）：
   - *故障原因分类（三级联动选择器）：
     * 一级：硬件故障 / 软件故障 / 通信故障 / 外部因素 / 人为损坏
     * 二级（以硬件为例）：充电模块 / 充电枪线 / 主控板 / 显示屏 / 电表 / 电子锁
     * 三级（以充电枪线为例）：枪头磨损 / 线缆破损 / 电子锁卡死 / CC/CP信号异常
   - *处理措施（多选标签 + 自定义文本补充）：
     * 更换备件 / 重启恢复 / 固件升级 / 参数重配 / 清洁维护 / 联系厂家 / 临时处置
   - *处理过程描述（多行文本输入，500字限制，支持语音转文字输入）。
   - *处理前照片（最多9张，带水印：时间+地点+工单号）：
     * 拍照界面带辅助框线提示（拍摄设备全景/故障部位/铭牌信息）。
   - *处理后照片（同样最多9张，带水印）。
   - 更换备件记录（可添加多条）：
     * 备件名称（下拉搜索选择，关联备件库存）+ 规格型号 + 数量 + 旧件回收确认。

4. 测试结果记录：
   - 修复后设备状态确认（单选）：恢复正常 / 部分恢复 / 未修复需返厂
   - 测试充电记录：是否进行了测试充电（是/否），测试充电时长和电量。
   - 遗留问题备注（如未完全修复，说明原因和后续计划）。

5. 底部操作栏（固定）：
   - [📞 请求远程协助] 按钮（视频通话连线技术专家）。
   - [↗ 升级工单] 按钮（P2升P1，或转给高级运维）。
   - [✅ 提交完工] 主按钮（蓝色，提交前校验所有必填项）。

【页面四：工单统计页（"我的"模块内）】
1. 时间范围切换：本周 / 本月 / 本季度。
2. 数据概览卡片（4个）：
   - 总处理工单数 | 平均响应时间 | 平均修复时间(MTTR) | SLA 达标率(%)
3. 工单类型分布饼图（故障/巡检/安装/投诉占比）。
4. 工单处理趋势折线图（按日/按周）。
5. 个人效率排名：在团队中的排名 + 与平均水平的对比雷达图（响应速度/修复速度/SLA达标率/用户满意度/工单量）。

要求：
- 工单数据支持离线暂存（创建/处理记录本地缓存，联网后自动同步，带冲突解决机制）。
- 拍照功能集成相机组件，强制添加防篡改水印（时间+GPS坐标+工单号+操作人）。
- 所有耗时操作（提交/上传）显示进度反馈（按钮Loading态 + 进度条）。
- 工单状态变更通过 WebSocket 实时推送，App 前台时 Toast 提示，后台时推送通知。
- 转派工单时，接收方需在 5 分钟内确认，超时自动回退并通知调度。
```

## 提示词 3：智能巡检全流程管理

> **定位**：定期/临时巡检任务的接收、现场巡检执行（含NFC/GPS打卡）、异常上报、巡检报告生成。

```text
请生成充电桩运维App的"智能巡检全流程管理"模块。
技术栈：UniApp + Vue 3 + Pinia + 高德/腾讯地图 SDK + NFC 插件。

【页面一：巡检任务列表（TabBar第三Tab）】
1. 顶部切换：
   - Tab：今日巡检任务 | 本周计划 | 历史巡检
   - 视图切换按钮：📋列表视图 / 🗺地图视图

2. 列表视图：
   - 按路线优化排序（根据当前定位，计算最优巡检路线顺序）。
   - 每项卡片：
     * 巡检任务编号 + 巡检类型标签（日常巡检 / 专项巡检 / 夜间巡检 / 雨季特巡）。
     * 站点名称 + 地址。
     * 巡检项数量（如 "共 25 项检查"）。
     * 计划巡检时间窗口（如 "08:00-12:00"）。
     * 进度指示（如 "已完成 0/3 站点" 进度条）。
     * 右侧：[开始巡检] 按钮。

3. 地图视图：
   - 地图显示所有今日待巡检站点标记。
   - 自动绘制推荐巡检路线（连线+箭头+序号）。
   - 标记颜色：灰色(待巡检) / 蓝色(进行中) / 绿色(已完成) / 红色(发现异常)。
```
