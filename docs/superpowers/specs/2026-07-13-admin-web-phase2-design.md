# EV充电平台 admin-web Phase 2 业务页面深度开发设计

**日期**: 2026-07-13
**范围**: admin-web 12 个业务模块的完整交互开发

---

## 总体模式

每个模块标准包含：
1. **列表页** — 搜索栏 + 数据表格（排序/筛选）+ 分页 + 批量操作 + 导出按钮
2. **新增/编辑弹窗** — el-dialog + el-form + 表单验证 rules
3. **详情弹窗/抽屉** — el-drawer + el-descriptions
4. **状态操作** — el-message-box 确认弹窗
5. **空状态 / Loading / 错误处理**

---

## 模块 2: 站点管理

### 列表页
**搜索字段**: 关键词（名称/地址/编号）、状态（运营中/已停运/维护中）、城市
**表格列**: 编号、名称、地址、设备数(在线/总数)、电价、服务费、状态、操作
**操作按钮**: 编辑、状态切换（启用/停运）、删除
**批量操作**: 批量停运、批量导出

### 新增/编辑弹窗
**表单字段**:
| 字段 | 类型 | 验证 |
|------|------|------|
| 充电站名称 | el-input | 必填，2-50字符 |
| 编号 | el-input | 必填，格式 BJ-CY-001 |
| 省份 | el-select | 必填 |
| 城市 | el-select | 必填，联动省份 |
| 区县 | el-input | 必填 |
| 详细地址 | el-input | 必填 |
| 经度 | el-input-number | 必填，73-135 |
| 纬度 | el-input-number | 必填，3-53 |
| 联系人 | el-input | 必填 |
| 联系电话 | el-input | 必填，手机格式 |
| 电价(元/kWh) | el-input-number | 必填，0-10，2位小数 |
| 服务费(元/kWh) | el-input-number | 必填，0-10，2位小数 |
| 停车费(元/h) | el-input-number | 0-100 |
| 营业时间 | el-input | 默认 00:00-24:00 |

### 详情抽屉
- 基本信息（el-descriptions，2列）
- 设备列表（el-table，该站点下所有设备）
- 充电记录（最近10条订单）

## 模块 3: 设备管理

### 列表页
**搜索字段**: 关键词（编号/OCPP ID）、所属站点、状态（在线/充电中/离线/故障）
**表格列**: 编号、所属站点、型号、类型(交/直流)、额定功率、实时功率、接口状态、最后心跳、操作
**操作按钮**: 详情、重置、解锁接口

### 详情弹窗
- 设备基本信息（el-descriptions）
- 充电接口列表（el-table，每个接口的状态/功率/交易ID）
- 最近告警（el-timeline）
- 远程操作按钮（重置/解锁/固件升级）

## 模块 4: 订单中心

### 列表页
**搜索字段**: 订单号、状态（充电中/已完成/退款中/异常/已取消）、充电站、时间范围
**表格列**: 订单号、用户(名+手机)、充电站、设备、电量、金额、时长、状态、开始时间、操作
**操作按钮**: 详情、退款（仅已完成状态）
**批量操作**: 批量导出

### 详情抽屉
- 订单信息（el-descriptions，含订单号/用户/站点/设备/接口）
- 充电数据（开始SOC/结束SOC/电量/功率曲线）
- 费用明细（电费/服务费/停车费/优惠/应付）
- 支付信息（支付方式/支付时间）
- 时间线（创建→开始充电→停止充电→结算→支付）

### 退款弹窗
- 退款金额（el-input-number，不超过应付金额）
- 退款原因（el-input，textarea，必填）

## 技术约束

- 所有表单使用 el-form + rules 验证
- 所有弹窗使用 el-dialog（宽600-800px）
- 详情使用 el-drawer（宽50%）或 el-dialog（宽700px）
- 所有表格支持排序（sortable）
- 所有金额显示使用 font-number 类 + toFixed(2)
- 所有状态使用 el-tag + 颜色映射
- 空状态使用 el-empty
- Loading 使用 v-loading 指令
